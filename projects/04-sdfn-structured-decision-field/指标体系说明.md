# SDFN 指标体系说明 / Metrics Guide

SDFN 日志中的指标不只用于看 AUC，也用于确认结构主链是否真的在工作。这个文件解释公开材料中出现的主要指标。

## 1. 排序效果指标

```text
auc_roc
```

本地评估使用的 ROC-AUC，是最直接的排序效果指标。384L8 / GraphExpand64 在 epoch 1 full evaluation 中：

```text
auc_roc = 0.846524
eval_rows_seen = 204306
eval_batches_seen = 400
```

```text
pairwise_batch_auc_proxy
```

训练或评估批次内的 pairwise 排序代理指标，用来观察 pairwise AUC objective 是否正在拉开正负样本排序。它不是最终官方指标，但能反映训练过程中的排序动力学。

```text
pairwise_auc_loss
pairwise_auc_loss_weight
pairwise_temperature
```

pairwise AUC loss 是主优化压力。当前主线中：

```text
pairwise_auc_loss_weight = 0.80
pairwise_temperature = 1.50
bce_loss_weight = 0.02
```

这说明训练主要优化样本间排序关系，BCE 只保留较小权重用于基础监督和概率标定。

## 2. 输出与结构纯化指标

```text
rank_score / base_rank_score
probability / final_probability / base_probability
```

`rank_score` 是最终排序 logit，`probability` 是 sigmoid 后的概率。在 GraphOnlyReadout 中，最终输出应来自图传播后的 well token states。

```text
final_base_rank_corr
final_base_prob_corr
prob_delta_abs
li_delta
li_rank
li_prob
```

这些指标用于确认是否有旁路参与最终出分。结构纯化版本中应看到：

```text
final_base_rank_corr = 1
final_base_prob_corr ≈ 1
prob_delta_abs = 0
li_delta = 0
li_rank = 0
li_prob = 0
```

含义是：最终分数没有被 LI residual、late fusion 或额外 probability correction 改写。

## 3. 图结构与 HLPO 指标

```text
adj_effective_density
```

有效邻接密度，表示经过 HLPO / permeability 后实际参与传播的结构密度。384L8 epoch 1：

```text
train summary: adj_effective_density = 0.212989
eval summary:  adj_effective_density = 0.160873
```

评估阶段因为 `hlpo_eval_permeability=0.55`，有效密度更低。

```text
hlpo_edge_gate_raw
hlpo_edge_gate
hlpo_token_gate
hlpo_permeability
```

`hlpo_edge_gate_raw` 是模型学到的原始 edge 参与质量；`hlpo_edge_gate` 是经过 permeability 后的实际 edge gate；`hlpo_token_gate` 是 token 参与质量；`hlpo_permeability` 控制结构开放程度。

当 `edge_raw` 与 `edge_gate` 分离时，说明结构选择信号已经被 permeability 收紧，而不是全量传播。

```text
effective_edge_mass
polar_edge_keep_mean
```

这些指标用于观察被保留下来的 edge 质量和规模，不直接等价于 AUC，但能帮助判断结构稀疏是否退化为空洞。

## 4. 波包与场变量指标

```text
bg_G
bg_V
bg_kappa
```

背景场变量。它们是从样本上下文中学习出来的结构环境，不是手写物理量：

- `G` 调制几何距离。
- `V` 类似场阈值 / 阻尼代理。
- `kappa` 表示场强 / 尺度代理。

```text
fast_mass
fast_momentum
phase_band
phase_reach_corr
```

这些指标用于观察 fast token 与 phase / reachability 的状态。它们不直接出分，而是参与图构造、edge visibility 和结构传播。

```text
learn_role_bias_std
role_pair_summary_*_rank_corr
```

role bias 和 role-pair summary 用于观察角色关系是否被模型学出差异。在 GraphOnlyReadout 中，role-pair 不作为最终 readout shortcut，但可以进入 graph construction 与 propagation。

## 5. 内部动力学指标

```text
internal_action
internal_resistance
internal_flow
internal_inertia
internal_bias
```

这些来自 InternalSampleDynamics，用于为图传播提供样本内部动力学 bias。它们不直接决定最终分数，但会影响 reader 的结构传播。

在公开解读中，它们可以理解为：模型不是只做静态图传播，还给当前样本生成一组内部状态调制量，用来影响 token 间的信息流动。

## 6. 性能与可执行性指标

```text
eval_time_sec
eval_rows_seen
eval_rows_per_sec
```

用于观察本地评估吞吐。384L8 epoch 1 full evaluation：

```text
eval_time_sec = 1570.746777
eval_rows_seen = 204306
eval_rows_per_sec = 130.069342
```

这些指标解释了为什么 384L8 Fullpath 在最终推理侧接近或超过时间约束，也解释了为什么需要 HLPO HardSkip 来落地更大图结构。

## 7. 阅读建议

公开解读 SDFN 时，优先看三类指标：

1. **效果指标**：`auc_roc`、`pairwise_batch_auc_proxy`。
2. **结构纯化指标**：`prob_delta_abs=0`、`li_delta=0`、`final_base_rank_corr≈1`。
3. **结构参与指标**：`adj_effective_density`、`hlpo_edge_gate/raw`、`hlpo_token_gate`、`hlpo_permeability`。

这三类指标合起来，才能说明 SDFN 的结果不是简单旁路修正，也不是只看 AUC 的黑盒结果，而是来自结构主链的可学习性验证。
